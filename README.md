# Finding Assistant — LangChain v1 & LangGraph

A **Finding Assistant** built on the modern LangChain v1 + LangGraph stack. It
answers IT Service Desk questions from `data/BonBon FAQ.pdf` using a RAG
pipeline, extracts a structured *Finding*, uses tools, keeps thread-based
short-term memory, and re-expresses the whole flow as an explicit LangGraph
workflow with a groundedness quality gate.

- **Assignment 1** — RAG agent: [`src/agent.py`](src/agent.py)
- **Assignment 2** — LangGraph workflow: [`src/graph.py`](src/graph.py)

---

## Architecture

```
data/BonBon FAQ.pdf
   │  loaders.py (pypdf)        → one Document per page
   │  splitters.py              → RecursiveCharacterTextSplitter chunks
   │  vector_store.py           → Chroma (persisted to .chroma/)
   │  retriever.py              → similarity retriever + context formatter
   ├── Assignment 1: agent.py   → create_react_agent + tools + InMemorySaver memory
   │                              + with_structured_output(Finding)
   └── Assignment 2: graph.py   → StateGraph: classify → [retrieve] → generate
                                  → extract_finding → validate_groundedness → report
```

**Modern patterns used** :

| Requirement | Implementation |
|---|---|
| Structured output | `llm.with_structured_output(Finding)` — no `LLMChain` |
| Agent | `create_agent` (LangChain v1) — no `AgentExecutor` |
| Short-term memory | `InMemorySaver` checkpointer keyed by `thread_id` — no `ConversationBufferMemory` |
| PDF loading | `pypdf` directly — avoids `langchain-community` |

## Setup

```bash
# 1. Create and activate a virtual environment
python -m venv .venv
# Windows (PowerShell):
.venv\Scripts\Activate.ps1
# macOS/Linux:
source .venv/bin/activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Configure credentials
cp .env.example .env      # then edit .env with your real Azure OpenAI values
```

### Environment variables (`.env`)

| Variable | Purpose |
|---|---|
| `AZURE_OPENAI_API_KEY` | Azure OpenAI key |
| `AZURE_OPENAI_ENDPOINT` | e.g. `https://your-resource.openai.azure.com/` |
| `AZURE_OPENAI_DEPLOYMENT_NAME` | Chat model **deployment** name |
| `AZURE_OPENAI_API_VERSION` | e.g. `2024-10-21` |
| `AZURE_OPENAI_EMBEDDING_DEPLOYMENT` | Embedding model **deployment** name |
| `LANGSMITH_API_KEY` | *(optional)* enables tracing automatically |
| `LANGSMITH_PROJECT` | *(optional)* trace project name |

> Deployment names are what you named your models in the Azure portal — not the
> base model ids. Never commit a real `.env` (it is git-ignored).

## How to run

```bash
# Build the vector index once (also runs automatically on first use)
python -c "from src.vector_store import build_vectorstore; build_vectorstore()"

# Assignment 1 — RAG agent demo (grounded Q&A, finding, memory)
python -m src.agent

# Assignment 2 — LangGraph workflow demo (+ writes outputs/graph.png)
python -m src.graph
```

Notebooks (interactive walkthroughs):

```bash
jupyter notebook notebooks/demo.ipynb            # Assignment 1
jupyter notebook notebooks/langgraph_demo.ipynb  # Assignment 2
```

## Rubric mapping 

| Area | Where |
|---|---|
| Document ingestion & chunking | [`loaders.py`](src/loaders.py), [`splitters.py`](src/splitters.py) |
| Embedding & retrieval | [`vector_store.py`](src/vector_store.py), [`retriever.py`](src/retriever.py) |

| Tool usage | [`tools.py`](src/tools.py) `search_document` (+ 2 more) |
| Short-term memory | `InMemorySaver` + `thread_id` in [`agent.py`](src/agent.py) / [`graph.py`](src/graph.py) |
| LangGraph workflow | [`graph.py`](src/graph.py) `build_graph` (6 nodes, conditional edge) |
| Validation & quality gate | [`graph.py`](src/graph.py) `validate_groundedness` |
| Documentation & usability | this README + `requirements.txt` + `.env.example` |
